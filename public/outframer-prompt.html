The Outframer Prompt

When generating plans, PRDs, or documentation in markdown, use this format so it can be shared cleanly with Outframer.

Prompt

Generate the output strictly in clean Markdown. Use clear headings, bullet points, and code blocks where appropriate. Avoid emojis or decorative formatting. This markdown will be shared using Outframer.

Example Markdown Output

# Product Requirements Document ## Overview This document outlines the requirements for the new feature. ## Goals - Increase user engagement - Improve conversion rates - Reduce support tickets ## Technical Requirements - API endpoint must support pagination - Response time < 200ms - Authentication via OAuth 2.0 ## Timeline - Phase 1: Research (2 weeks) - Phase 2: Development (4 weeks) - Phase 3: Testing (1 week)
